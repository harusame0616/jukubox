# テスト共通ルール

- 実装の詳細のテストを避け、振る舞いのテストを優先する
  - href, className などの DOM 属性値の assert は避ける
  - 内部で特定の関数が呼び出されたかどうか（モック関数のスパイなど）の検証は避ける
  - ユーザーや呼び出し側から見える結果（戻り値、表示されるテキスト、状態変化）を起点にテストする
- なるべく上位からの結合テストを優先し、極力 mock を避ける
  - 基本は medium テストで担保する
  - medium で再現が困難な以下のケースのみ small テストで mock を使って補完する
    - 外部 API が 500 を返す、ネットワーク異常などのエッジケース
    - 通常のユースケースでは発生しない経路（防御的なエラー分岐、exhaustive check の default 句など）
