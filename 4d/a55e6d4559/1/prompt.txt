GitHub Advanced Security（ボット） 16分前 重要度：高 警告 CodeQL / 機密データに対する強度の低い暗号ハッシュアルゴリズムの使用

機密データ（パスワード）のハッシュ化にSHA256が使用されています。このアルゴリズムは計算負荷が低く、パスワードの保護用としては安全ではありません。

詳細を表示
@apps/api/routes/users/settings/api-keys/hashed-api-key.entity.go 

API Key の生成でパスワードと違って文字数も多く強度が高いかつ API の検証で頻繁にでコードもするため sha256 が適切だと思うのですがよくない？
