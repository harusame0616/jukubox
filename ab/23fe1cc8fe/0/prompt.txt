@.github/workflows/api-small-test.yml  

現在 changes と api-small-test の２つのジョブがある。
changes では対象のパスが変更されているかをチェックテスト実施の必要があるかを確認している
その後変更があれば api-small-test を実行している

元々の意図はコメントに書いてあるとおり PR のマージ条件で api-small-test の実行を必須にしたいが、
ci の path filter を使って実行しないようにしてしまうと該当のパスを修正しなかった場合、
ワークフローが起動せずマージできなくなってしまうため、ジョブでチェックするようにしている

ただし、PR のマージ条件として指定できるのはジョブなため結局現在のようにジョブが別れてしまっていると
api-small-test を指定した時に skip されてしまって通らない。
逆に changes を指定するとテストが通っても通らなくてもマージできてしまう。
そのため changes と api-small-test を１つのジョブに統合する
