# API仕様書: トピック詳細取得 (Get Topic Detail)

## 1. 概要・エンドポイント定義

コース内の特定トピックの本文 (講義テキスト) と概要を取得します。エージェントが講義を進めるための知識ソースとして使用します。

| 項目       | 内容                                                                            |
| :--------- | :------------------------------------------------------------------------------ |
| **Method** | `GET`                                                                           |
| **Path**   | `/v1/courses/{courseId}/sections/{sectionId}/topics/{topicId}`                  |
| **認証**   | 要 (API キー / Bearer Token)                                                    |

---

## 2. リクエスト

### 2.1. パスパラメータ

| パラメータ名 | 型   | 必須 | 説明                          |
| :----------- | :--- | :--- | :---------------------------- |
| `courseId`   | UUID | Yes  | 対象コース ID。               |
| `sectionId`  | UUID | Yes  | 対象セクション ID。           |
| `topicId`    | UUID | Yes  | 対象トピック ID。             |

### 2.2. クエリパラメータ

| パラメータ名 | 型   | 必須 | 説明                                                                                    |
| :----------- | :--- | :--- | :-------------------------------------------------------------------------------------- |
| `userId`     | UUID | No   | 閲覧者のユーザー ID。下書き / 限定公開コースなどで著者本人として閲覧する際に使用する。  |

### 2.3. リクエストボディ

なし。

---

## 3. レスポンス

### 3.1. 成功時 (`200 OK`)

```json
{
  "courseId":    "11111111-1111-1111-1111-111111111111",
  "sectionId":   "22222222-2222-2222-2222-222222222222",
  "topicId":     "33333333-3333-3333-3333-333333333333",
  "title":       "変数と型",
  "description": "TypeScript の基本的な変数定義と型注釈について学びます。",
  "content":     "# 変数と型\n\nTypeScript では..."
}
```

| フィールド    | 型     | 説明                                                              |
| :------------ | :----- | :---------------------------------------------------------------- |
| `courseId`    | UUID   | コース ID。                                                       |
| `sectionId`   | UUID   | セクション ID。                                                   |
| `topicId`     | UUID   | トピック ID。                                                     |
| `title`       | String | トピックタイトル。                                                |
| `description` | String | トピックの概要 (1〜数行)。                                        |
| `content`     | String | 講義本文 (Markdown 等のテキスト)。                                |

### 3.2. エラー時

| HTTP Status                   | エラーコード (code)        | 発生条件                                                                    |
| :---------------------------- | :------------------------- | :-------------------------------------------------------------------------- |
| **400 Bad Request**           | `INPUT_VALIDATION_ERROR`   | `courseId` / `sectionId` / `topicId` / `userId` のいずれかが UUID でない。  |
| **401 Unauthorized**          | `UNAUTHORIZED`             | API キー欠落 / 無効。                                                       |
| **404 Not Found**             | `TOPIC_DETAIL_NOT_FOUND`   | 指定された `(courseId, sectionId, topicId)` の組み合わせが存在しない、または閲覧権限がない場合。 |
| **500 Internal Server Error** | `INTERNAL_ERROR`           | DB エラーなど予期しないサーバーエラー。                                     |

エラーレスポンス例:

```json
{
  "code":    "TOPIC_DETAIL_NOT_FOUND",
  "message": "Topic detail is not found"
}
```

---

## 4. 振る舞い・ロジック

- 受講中であるか否かに依存せず、コース閲覧権限がある限り取得可能 (公開コース、または `userId` が著者の下書き / 非公開コース)。
- 「次に学ぶべきトピック」の `(sectionId, topicId)` は [受講中詳細.md](./受講中詳細.md) の `nextTopic` から取得し、その値をそのまま本エンドポイントのパスに渡す想定。
