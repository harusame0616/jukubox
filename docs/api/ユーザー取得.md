# API仕様書: ユーザー取得 (Get Me)

## 1. 概要・エンドポイント定義

ログイン中ユーザー自身のプロフィール情報 (ニックネーム / 自己紹介) を取得します。

| 項目       | 内容                          |
| :--------- | :---------------------------- |
| **Method** | `GET`                         |
| **Path**   | `/v1/me`                      |
| **認証**   | 要 (API キー / Bearer Token)  |

---

## 2. リクエスト

### 2.1. パスパラメータ

なし。対象ユーザーは Authorization ヘッダーから解決します。

### 2.2. リクエストボディ

なし。

---

## 3. レスポンス

### 3.1. 成功時 (`200 OK`)

```json
{
  "nickname":  "ジュク太郎",
  "introduce": "Web エンジニア。 Go と TypeScript を学んでいます。"
}
```

| フィールド  | 型     | 説明                                  |
| :---------- | :----- | :------------------------------------ |
| `nickname`  | String | ユーザーの表示名。 1〜50 文字。       |
| `introduce` | String | 自己紹介。 0〜500 文字。              |

### 3.2. エラー時

| HTTP Status                   | エラーコード (code)        | 発生条件                                  |
| :---------------------------- | :------------------------- | :---------------------------------------- |
| **401 Unauthorized**          | `UNAUTHORIZED`             | API キー欠落 / 無効。                     |
| **404 Not Found**             | `USER_NOT_FOUND`           | 認証ユーザーに対応するレコードが存在しない。 |
| **500 Internal Server Error** | `SERVER_INTERNAL_ERROR`    | DB エラーなど予期しないサーバーエラー。   |

---

## 4. 認証 (Authentication)

API キー (Bearer) 認証で保護されています。詳細は [認証.md](./認証.md) を参照してください。

---

## 5. リクエスト例 (cURL)

```bash
curl -X GET "https://api.example.com/v1/me" \
     -H "Authorization: Bearer sk_live_abc123def456..."
```
