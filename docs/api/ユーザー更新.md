# API仕様書: ユーザー更新 (Update Me)

## 1. 概要・エンドポイント定義

ログイン中ユーザー自身のプロフィール (ニックネーム / 自己紹介) を更新します。

| 項目       | 内容                                  |
| :--------- | :------------------------------------ |
| **Method** | `PATCH`                               |
| **Path**   | `/v1/me`                              |
| **認証**   | 要 (Supabase JWT / Bearer Token)      |

---

## 2. リクエスト

### 2.1. パスパラメータ

なし。対象ユーザーは Authorization ヘッダーの JWT から解決します。

### 2.2. リクエストボディ (JSON)

| パラメータ名 | 型     | 必須 | 説明                                |
| :----------- | :----- | :--- | :---------------------------------- |
| `nickname`   | String | Yes  | 表示名。 1〜50 文字 (符号位置単位)。 |
| `introduce`  | String | Yes  | 自己紹介。 0〜500 文字 (符号位置単位)。 |

`PATCH` であってもリクエストボディは「更新後の値」をすべて指定します (現状の API 実装では部分更新ではなく全項目の上書き)。

---

## 3. レスポンス

### 3.1. 成功時 (`200 OK`)

```json
{
  "nickname":  "ジュク太郎",
  "introduce": "Web エンジニア。 Go と TypeScript を学んでいます。"
}
```

| フィールド  | 型     | 説明                |
| :---------- | :----- | :------------------ |
| `nickname`  | String | 更新後のニックネーム。 |
| `introduce` | String | 更新後の自己紹介。     |

### 3.2. エラー時

| HTTP Status                   | エラーコード (code)        | 発生条件                                                                  |
| :---------------------------- | :------------------------- | :------------------------------------------------------------------------ |
| **400 Bad Request**           | `INPUT_VALIDATION_ERROR`   | ボディが不正な JSON、 `nickname` の長さが範囲外、 `introduce` が 500 文字超。 |
| **401 Unauthorized**          | `UNAUTHORIZED`             | Authorization ヘッダー欠落、または JWT が無効な場合。                     |
| **404 Not Found**             | `USER_NOT_FOUND`           | 認証ユーザーに対応するレコードが存在しない。                              |
| **500 Internal Server Error** | `SERVER_INTERNAL_ERROR`    | DB エラーなど予期しないサーバーエラー。                                   |

エラーレスポンス例:

```json
{
  "code":    "INPUT_VALIDATION_ERROR",
  "message": "validation error: nickname must be between 1 and 50 characters"
}
```

---

## 4. 振る舞い・ロジック

- `nickname` の長さは 1〜50 (Unicode 符号位置)。範囲外で `400 INPUT_VALIDATION_ERROR`。
- `introduce` は最大 500 (Unicode 符号位置)。超過で `400 INPUT_VALIDATION_ERROR`。
- 認証は Supabase JWT で行い、 JWT の `sub` をそのまま対象ユーザー ID として使用。

---

## 5. 認証 (Authentication)

本エンドポイントは Supabase の JWT を用いた認証で保護されており、 [認証.md](./認証.md) で説明している API キー (Bearer) 認証とは別系統です。

---

## 6. リクエスト例 (cURL)

```bash
curl -X PATCH "https://api.example.com/v1/me" \
     -H "Authorization: Bearer eyJhbGciOi..." \
     -H "Content-Type: application/json" \
     -d '{
           "nickname":  "ジュク太郎",
           "introduce": "Web エンジニア"
         }'
```
