# API仕様書: 受講進捗更新 (Update Topic Progress)

## 1. 概要・エンドポイント定義

ログイン中ユーザーのコース内の特定トピックに対して、進捗ステータスを更新します。エージェントが「学習開始 (`IN_PROGRESS`)」「学習完了 (`COMPLETED`)」のタイミングで呼び出します。

| 項目       | 内容                                          |
| :--------- | :-------------------------------------------- |
| **Method** | `PATCH`                                       |
| **Path**   | `/v1/me/enrollments/{courseId}`               |
| **認証**   | 要 (API キー / Bearer Token)                  |

---

## 2. リクエスト

### 2.1. パスパラメータ

| パラメータ名 | 型   | 必須 | 説明                  |
| :----------- | :--- | :--- | :-------------------- |
| `courseId`   | UUID | Yes  | 対象コース ID。       |

### 2.2. リクエストボディ (JSON)

| パラメータ名 | 型     | 必須 | 説明                                              |
| :----------- | :----- | :--- | :------------------------------------------------ |
| `topicId`    | UUID   | Yes  | 進捗を更新するトピック ID。                       |
| `status`     | String | Yes  | 新しい進捗ステータス。 `IN_PROGRESS` または `COMPLETED`。 |

`NOT_STARTED` を直接指定することはできません (進捗レコードを作らずに自然に取り扱われる初期値です)。

---

## 3. レスポンス

### 3.1. 成功時 (`200 OK`)

```json
{
  "topicId": "44444444-4444-4444-4444-444444444444",
  "status":  "COMPLETED"
}
```

| フィールド | 型     | 説明                       |
| :--------- | :----- | :------------------------- |
| `topicId`  | UUID   | 更新対象のトピック ID。    |
| `status`   | String | 更新後のステータス。       |

### 3.2. エラー時

| HTTP Status                   | エラーコード (code)        | 発生条件                                                                                     |
| :---------------------------- | :------------------------- | :------------------------------------------------------------------------------------------- |
| **400 Bad Request**           | `INPUT_VALIDATION_ERROR`   | UUID 形式不正、 `status` が `IN_PROGRESS` / `COMPLETED` 以外、状態遷移が不正、など。         |
| **401 Unauthorized**          | `UNAUTHORIZED`             | API キー欠落 / 無効。                                                                        |
| **404 Not Found**             | `COURSE_NOT_FOUND`         | 指定された `courseId` のコースが存在しない場合。                                             |
| **404 Not Found**             | `TOPIC_NOT_FOUND`          | 指定された `topicId` がコース内に存在しない場合。                                            |
| **404 Not Found**             | `NOT_ENROLLED`             | 対象コースを受講していない (Enrollment 未作成) 場合。                                        |
| **500 Internal Server Error** | `SERVER_INTERNAL_ERROR`    | DB エラーなど予期しないサーバーエラー。                                                      |

エラーレスポンス例:

```json
{
  "code":    "INPUT_VALIDATION_ERROR",
  "message": "invalid progress status transition"
}
```

---

## 4. 振る舞い・ロジック

### 4.1. ステータス遷移

| 現在 \ 次 | `IN_PROGRESS` | `COMPLETED` |
| :--------- | :------------ | :---------- |
| (進捗なし / `NOT_STARTED`) | 可 (新規作成) | 可 (新規作成) |
| `IN_PROGRESS` | 可 (冪等)     | 可          |
| `COMPLETED`   | **不可** (`INPUT_VALIDATION_ERROR`) | 可 (冪等)   |

`COMPLETED` から `IN_PROGRESS` への巻き戻しは禁止されており、 `400 INPUT_VALIDATION_ERROR` (`invalid progress status transition`) を返します。

### 4.2. 進捗レコードの扱い

- 該当トピックの進捗レコードが無ければ新規作成、あれば更新。
- 受講開始 (`POST /v1/me/enrollments`) を行っていないコースに対しては `404 NOT_ENROLLED`。

---

## 5. 認証 (Authentication)

API キー (Bearer) 認証で保護されています。詳細は [認証.md](./認証.md) を参照してください。

---

## 6. リクエスト例 (cURL)

```bash
# 学習開始
curl -X PATCH "https://api.example.com/v1/me/enrollments/11111111-1111-1111-1111-111111111111" \
     -H "Authorization: Bearer sk_live_abc123def456..." \
     -H "Content-Type: application/json" \
     -d '{
           "topicId": "44444444-4444-4444-4444-444444444444",
           "status":  "IN_PROGRESS"
         }'

# 学習完了
curl -X PATCH "https://api.example.com/v1/me/enrollments/11111111-1111-1111-1111-111111111111" \
     -H "Authorization: Bearer sk_live_abc123def456..." \
     -H "Content-Type: application/json" \
     -d '{
           "topicId": "44444444-4444-4444-4444-444444444444",
           "status":  "COMPLETED"
         }'
```
