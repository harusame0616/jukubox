# API仕様書: 受講開始 (Enroll Course)

## 1. 概要・エンドポイント定義

ログイン中ユーザー (`/v1/me`) が指定したコースの受講を開始します。受講開始 (Enrollment) を新規作成し、その識別情報を返却します。トピックごとの進捗は別 API ([受講進捗更新.md](./受講進捗更新.md)) で更新します。

| 項目       | 内容                          |
| :--------- | :---------------------------- |
| **Method** | `POST`                        |
| **Path**   | `/v1/me/enrollments`          |
| **認証**   | 要 (API キー / Bearer Token)  |

---

## 2. リクエスト

### 2.1. パスパラメータ

なし。受講主体は認証されたユーザー (API キーから解決) です。

### 2.2. リクエストボディ (JSON)

| パラメータ名 | 型   | 必須 | 説明                 |
| :----------- | :--- | :--- | :------------------- |
| `courseId`   | UUID | Yes  | 受講対象のコース ID  |

---

## 3. レスポンス

### 3.1. 成功時 (`201 Created`)

```json
{
  "courseId":   "22222222-2222-2222-2222-222222222222",
  "enrolledAt": "2026-05-01T10:00:00Z"
}
```

| フィールド名 | 型     | 説明                       |
| :----------- | :----- | :------------------------- |
| `courseId`   | UUID   | 受講開始したコース ID      |
| `enrolledAt` | String | 受講開始時刻 (RFC 3339)    |

### 3.2. エラー時 (4xx)

| HTTP Status              | エラーコード (code)       | 発生条件                                                                       |
| :----------------------- | :------------------------ | :----------------------------------------------------------------------------- |
| **400 Bad Request**      | `INPUT_VALIDATION_ERROR`  | `courseId` が空、または UUID 形式でない場合。ボディが不正な JSON。             |
| **403 Forbidden**        | `ENROLLMENT_FORBIDDEN`    | 公開コースでない、かつコース著者本人でもない場合。                             |
| **404 Not Found**        | `COURSE_NOT_FOUND`        | 指定された `courseId` のコースが存在しない場合。                               |
| **409 Conflict**         | `ALREADY_ENROLLED`        | 既に同コースを受講中の場合。                                                   |

**エラーレスポンス例:**

```json
{
  "code":    "ALREADY_ENROLLED",
  "message": "already enrolled in this course"
}
```

---

## 4. 振る舞い・ロジック

- 同一ユーザー × 同一コースの組み合わせで Enrollment は最大 1 件のみ。重複作成は `409 ALREADY_ENROLLED` を返す。
- 受講開始可否のチェックは Course 集約側で行う (公開ステータス、または著者本人のみ)。
- Enrollment 作成のみを行い、トピックの進捗レコードは作成しない。トピックの進捗開始 / 完了は [受講進捗更新.md](./受講進捗更新.md) で行う。

---

## 5. 認証 (Authentication)

API キー (Bearer) 認証で、リクエスト元ユーザーが受講主体になります。詳細は [認証.md](./認証.md) を参照してください。

---

## 6. リクエスト例 (cURL)

```bash
curl -X POST "https://api.example.com/v1/me/enrollments" \
     -H "Authorization: Bearer sk_live_abc123def456..." \
     -H "Content-Type: application/json" \
     -d '{
           "courseId": "22222222-2222-2222-2222-222222222222"
         }'
```
