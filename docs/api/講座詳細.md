# API仕様書: 講座詳細 (Get Course Detail)

## 1. 概要・エンドポイント定義

公開済み講座 (`publish_status = published` かつ `visibility = public`) の詳細を `(authorSlug, courseSlug)` で取得します。未ログインでも閲覧可能で、ログインユーザーには受講中フラグ (`isEnrolled`) を含めて返します。

| 項目       | 内容                                                |
| :--------- | :-------------------------------------------------- |
| **Method** | `GET`                                               |
| **Path**   | `/v1/courses/{authorSlug}/{courseSlug}`             |
| **認証**   | 任意（Authorization があれば検証して isEnrolled を出す） |

---

## 2. リクエスト

### 2.1. パスパラメータ

| パラメータ名  | 型     | 必須 | 説明              |
| :------------ | :----- | :--- | :---------------- |
| `authorSlug`  | String | Yes  | 著者の URL slug   |
| `courseSlug`  | String | Yes  | コースの URL slug |

### 2.2. リクエストボディ

なし。

---

## 3. レスポンス

### 3.1. 成功時 (`200 OK`)

```json
{
  "courseId": "22222222-2222-2222-2222-222222222222",
  "title": "Go 入門",
  "description": "Go 言語の基礎を学ぶコース",
  "slug": "go-basics",
  "tags": ["go", "backend"],
  "author": {
    "authorId": "11111111-1111-1111-1111-111111111111",
    "name": "Alice",
    "slug": "alice"
  },
  "sections": [
    {
      "sectionId": "33333333-3333-3333-3333-333333333333",
      "title": "はじめに",
      "description": "Go 言語の概要と環境構築",
      "topics": [
        {
          "topicId": "44444444-4444-4444-4444-444444444444",
          "title": "Hello World",
          "description": "最初のプログラム"
        }
      ]
    }
  ],
  "isEnrolled": false
}
```

| フィールド名               | 型      | 説明                                                                |
| :------------------------- | :------ | :------------------------------------------------------------------ |
| `courseId`                 | UUID    | 講座 ID（参考用。受講開始 API では使わない）                        |
| `title`                    | String  | 講座タイトル                                                        |
| `description`              | String  | 講座概要                                                            |
| `slug`                     | String  | 講座 URL slug                                                       |
| `tags`                     | Array&lt;String&gt; | 講座に紐付くタグの配列                                  |
| `author.authorId`          | UUID    | 著者 ID                                                             |
| `author.name`              | String  | 著者名                                                              |
| `author.slug`              | String  | 著者 URL slug                                                       |
| `sections[].sectionId`     | UUID    | セクション ID                                                       |
| `sections[].title`         | String  | セクションタイトル                                                  |
| `sections[].description`   | String  | セクション概要                                                      |
| `sections[].topics[]`      | Array   | トピック一覧（順序保証）                                            |
| `topics[].topicId`         | UUID    | トピック ID                                                         |
| `topics[].title`           | String  | トピックタイトル                                                    |
| `topics[].description`     | String  | トピック概要                                                        |
| `isEnrolled`               | Boolean | 認証済みユーザーがそのコースを受講中なら `true`、それ以外は `false` |

### 3.2. エラー時 (4xx / 5xx)

| HTTP Status              | エラーコード (errorCode)  | 発生条件                                                                                       |
| :----------------------- | :------------------------ | :--------------------------------------------------------------------------------------------- |
| **400 Bad Request**      | `INPUT_VALIDATION_ERROR`  | `authorSlug` / `courseSlug` が空。                                                              |
| **404 Not Found**        | `COURSE_NOT_FOUND`        | `(authorSlug, courseSlug)` の組に該当する講座が存在しない、または `published` & `public` でない。 |
| **500 Internal Server Error** | `SERVER_INTERNAL_ERROR` | サーバー内部エラー。                                                                          |

未ログイン時でも、講座が公開済みであれば 200 を返します。`isEnrolled` は常に `false` になります。

---

## 4. 振る舞い・ロジック

- `(authorSlug, courseSlug)` で `authors` と `courses` を JOIN して 1 講座を取得する。
- 取得した講座の `publish_status != 'published'` または `visibility != 'public'` の場合は 404 を返す（未ログインからは下書き・限定公開を区別なく非公開）。
- Authorization ヘッダーが付与されていれば JWT / API キーで認証を試み、成功したユーザーの `enrollments` を確認して `isEnrolled` を返す。
- 認証ヘッダーが不正でも 401 にはせず、 `isEnrolled = false` として 200 を返す（公開ページのため）。
- セクションとトピックは `index ASC` 順で返す。

---

## 5. 認証 (Authentication)

認証は任意です。詳細は [認証.md](./認証.md) を参照してください。

---

## 6. リクエスト例 (cURL)

```bash
# 未ログイン
curl "https://api.example.com/v1/courses/alice/go-basics"

# ログイン中
curl "https://api.example.com/v1/courses/alice/go-basics" \
     -H "Authorization: Bearer sk_live_abc123def456..."
```
