  3. Vercel Lab emulate: ローカル OAuth シミュレーション方法の具体的な手順

  1. DB マイグレーション (apps/api/migrations/000005_users.up.sql)

  - routes/auth/（CQRS ではなく薄いハンドラ群）

    - callback.handler.go: GET /v1/auth/callback → code → Supabase でトークン交換 → users upsert → JWT を JSON で返す（Cookie は Next.js 側で設定）
    - logout.handler.go: POST /v1/auth/logout → Supabase セッション失効
  - main.go: ルート登録、DB pool に加え Supabase クライアント注入
  - テスト: small（JWT 検証）+ medium（handler、Supabase はフェイク）
